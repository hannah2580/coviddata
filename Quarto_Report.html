Data Exploration and Contextualization

This report utilizes the COVID-19 Cases and Deaths Database to examine how COVID-19 deaths and cases vary across location and time. The dataset provides daily, country-level statistics of confirmed COVID-19 deaths and cases, along with weekly and biweekly counts. From a social sciences perspective, this dataset is crucial for international organisations and governments in determining which countries to prioritize sending humanitarian aid to, depending on the severity of their COVID-19 statistics (World Health Organisation 2022). It is also crucial at a national level for monitoring case and death trends, enabling governments to assess policy effectiveness and respond to imminent outbreaks. (World Health Organisation 2021)

Function: Annual Case and Death Counts for Selected Country

I created the function summarise_totals() to automate the calculation of yearly totals for COVID-19 cases and deaths for any specified location. This function is useful as summarizing cases and deaths by year simplifies comparisons of COVID-19 trends between different locations. Automating this process enables quick cross-country comparisons without having to repeatedly write the code and also minimizes the risk of coding errors from manually computing these calculations.

summarise_totals <- function(data, country_name) {
  data %>%
    filter(location == country_name) %>%
    group_by(year) %>%
    summarise(annual_cases = sum(new_cases, na.rm = TRUE),
              annual_deaths = sum(new_deaths, na.rm = TRUE))
}

Data Visualization and Functional Programming

Function: Daily New Cases for Selected Country

I created a function plot_country_cases() to automate the plotting of daily new COVID-19 cases for any selected country. This function is important as it displays the trajectory and duration of outbreaks, which is a useful illustration for governments in understanding the effectiveness of certain policies in containing transmission. Automating this process also improves efficiency as it eliminates the need to repeatedly write the code for different countries.

plot_country_cases <- function(data, country_name) {
  data %>% 
    filter(location == country_name) %>%
    ggplot(aes(x = date, y = new_cases)) + 
    geom_line(color = "red") + 
    labs(title = country_name, 
         # paste() allows the variable country_name to be included in the string 
         x = "",
         y = "Daily New COVID-19 Cases") +  
    theme_minimal()
}

Data Visualization 1: Daily New Cases in High-income vs Lower-middle income countries

While the earlier wrangling results provided the annual case counts for high-income and lower-middle income countries, they do not reveal how cases evolved within each year. To visualize these trends more precisely over time, I used the country_plot_cases function to plot daily new cases for both groups and used grid.arrange() to display both graphs side-by-side for a clearer visual comparison of their trajectories across the 2020-2024 period.

higher_income <- plot_country_cases(data, "High-income countries")
lower_income <- plot_country_cases(data, "Lower-middle-income countries")
grid.arrange(higher_income, lower_income, ncol = 2)

Figure 1 indicates that both high-income and lower-middle income countries experienced multiple waves of cases, with the largest peaks occurring in 2021 and 2022, corresponding with the spread of the Omicron variant (Daria and Islam 2022). By 2023, both groups experienced a sharp decline in new cases, suggesting global vaccination roll outs or vaccine efficacy in containing the spread of the virus.

Data Visualization 2: Total Cases in China, France, India and United States

Earlier wrangling tasks revealed the United States, China, India and France ranked highest in terms of total number of COVID-19 cases. I computed a bar chart to visualize the relative magnitudes of cases between these countries, as they make the scale of variation immediately observable, which is less obvious from reading the numbers alone.

country_cases <- data %>%
  filter(location %in% c("United States","China","India", "France")) %>%
  group_by(location) %>%
  summarise(total_cases = sum(new_cases, na.rm = TRUE)) 

ggplot(country_cases, aes(x = location, y = total_cases))+
  geom_col(position = "dodge") +
  labs(x = "", 
       y = "Total COVID-19 Cases") +
  theme_minimal()

Figure 2 illustrates that the total case counts in the United States and China are substantially higher than India and France by around 50 million cases. This suggests that the spread of COVID-19 was more extensive in the United States and China. However, interpreting raw case counts can be misleading when comparing countries with large differences in population sizes, as larger populations may naturally report a higher absolute number of cases solely due to their size.To address this, the next visualization adjusts for population size by expressing the number of total cases per one million people, which provide a fairer comparison of scale of infection between the countries.

Data Visualization 3: Total Cases Per Million People in China, France, India, and the United States

The dataset did not contain country-level population data, thus, population values of the four countries were taken from the World Bank Open Data (World Bank Open Data, n.d.) and inputted into the calculations for total COVID-19 cases per million people.

# Input population values
population_lookup <- tibble(location = c("United States","China","India","France"),
                            population = c(340100000,1409000000,1451000000, 68520000))

# Compute total cases per country
pop_adjusted <- data %>%
  group_by(location) %>%
  summarise(total_cases = sum(new_cases, na.rm = TRUE)) %>%
  inner_join(population_lookup, by = "location") %>%
  mutate(cases_per_million = (total_cases / population) * 1e6)

ggplot(pop_adjusted, aes(x = location, y = cases_per_million)) +
  geom_col(position = "dodge") +
  labs(x = "",
       y = "Total COVID-19 Cases per Million People") +
  theme_minimal()

Figure 3 indicates that cases per million people in France are substantially higher, even though France reported the fewest absolute cases out of the four countries. This indicates that the spread of COVID-19 in France was more severe and widespread than in the other three countries. Contrastingly, India and China display a lower case count per million people, suggesting a lower infection intensity relative to the population size. Thus, expressing cases per million people enables a fairer comparison of the scale of infection between countries with vastly different population sizes.

Critical Engagement with AI: Copilot

Copilot was a useful tool throughout the coding process, particularly in helping me resolve errors in my code. When a code chunk failed to run, I could input both the code and error message into Copilot, which would provide step-by-step guidance of how to fix the code. For instance, it helped identify that my code failed to run due to a missing pipe operator or because I overwrote a column I was trying to call upon. More importantly, Copilot clearly explained how its suggested changes corrected the code, which improved my understanding of how R processes sequential operations.

Copilot was especially helpful for the visualization section. There were instances where my line graphs displayed vertical lines or returned blank outputs when I tried plotting total cases over time for a selected location. Copilot identified the that the issue stemmed from my x-axis being incorrectly set to location rather than date, and suggested the appropriate correction. This not only helped resolve the error but taught me the importance of using the correct variables within the aes() function.

However, there were instances where Copilot was inaccurate. I initially produced a single, overlapping line graph rather than two separate plots for Figure 1. When I gave my code to Copilot and asked why this was the case, it claimed that I had used geom_area() or geom_ribbon() in my previous code, which I had not. After several unsuccessful attempts to resolve the problem, I asked a new question of displaying side-by-side plots, and it recommended me to use the grid.arrange() function, which successfully produced the output.

Overall, Copilot was very useful in helping me resolve errors that arose. But it more so helped deepen my understanding of sequential processing in R and data visualization functions through its clear, step-by-step explanations.
